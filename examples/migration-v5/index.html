Migration From v4 (and below) to v5

Version 5 brings WebGL Renderer to the game. It is a default renderer from now on, and there might be possible considerations:

  • It might cause issues with (older) devices due to WebGL support.
  • Problems with tainted data, make sure you download images from the same domain, or set crossOriginPolicy: 'Anonymous'.
  • Tile data is not directly modifiable since it now lives on the GPU, therefore tile-drawing-based functions do not work.
  • In general, any data modification might be problematic, so plugins like OpenSeadragon Filtering will not likely work.

If you experience issues, you might consider sticking with the Context2D canvas drawer. We will be also grateful for any feedbacks on issues with devices. 

const viewer = OpenSeadragon({
    ....
    drawer: 'canvas'
});
